I - Introduction :

assembly is an ensemble of programming languages that represent machine code instructions as human readable code, which makes them the closest in nature to machine code, the first recorded use of this language was found in Kathleen and Andrew Donald Booth's 1947 work, [Coding for A.R.C](Kathleen and Andrew Donald Booth's 1947 work, Coding for A.R.C)

III - Memory Management :

1 - Data :

unlike other programming languages, assembly stores it's variables not in RAM but in the CPU in designated 32-bit registries.
most of these variables are prefixed like AX: the accumulator, you can name your own variable but we'll be sticking with the standard variable names for this Documentation.
most notable variables :

  • AX: the Primary Accumulator
    used mainly to preform arithmetic operations.
  • BX: Base Register
    used for storing data.
  • CX: Count Register
    reserved. used for storing loop count's in iterations.
  • DX: Data Register
    half-reserved more... used for storing data.

To be honest, although these Variables are what you would be using when programming assembly, it is actually not the full 32-bit register as I mentioned. the Actual full register names are EAX, EBX, ECX and EDX. for the purpose of this Doc, we only deal with data less then 32-bit, thus we don't need the full register.

2 - Memory Address :

to access a variables pointer, simply put the variable name in [], ex: [AX]

Note : even with in these sub-registers we have further sub-sub-register

These sub-registers are mainly hold-overs from older, 16-bit versions of the instruction set. However, they are sometimes convenient when dealing with data that are smaller than 32-bits (e.g. 1-byte ASCII characters). source->1

  • Note : CPU's with a 64-bit architecture have bigger registers for example : EAX would be a sub-register in the full 64-bit register RAX.
  • Note : the EDX register is said to be half-reserved because any results from operations preformed on the EAX that surpass 32-bit have their excess stored in 16-bits from EAX
    and 16-bits from EDX.

II - Operations :

for simplicity's sake we'll put some ground rules.

  • all operations take one line :
  • Variables Allowed for use are Ax, BX and DX.
  • all boiler plate is unessential.
  • MSW : refers to the machine status word, which is a registry that stores the results of some operations (like CMP) and gives the input to some instructions (like JMP<conditional>), due it's nature it must be used as an output right after being given an input.

1 - Operations :

any Operation works in this form :
[OPERATION] <OP1>, <OP2> OR [OPERATION] <OP>, Thus with that in mind the operations we'll be focusing in are these.
Operations for use :

  • MOV : copies the value from OP2 to OP1.
  • PUSH : Pushes OP to the top of the memory stack
  • POP : Pops the last element of the memory stack into OP
  • ADD : Adds OP2 to OP1
  • SUB : Subtracts OP2 from OP1
  • MUL : Multiplies OP1 by OP2
  • DIV : Divides OP1 by OP2
  • CMP : Compares between OP1 and OP2 and stores the result in MSW

2 - Conditions :

like any programming language assembly holds conditional structures, like <if>..<else>, <while> etc..
Conditional structures :

  • JMP : "jumps" towards OP (aka it's a goto function)
  • <conditional>JMP : the name of this condition changes with each intended use, but it simply preforms an JMP if the condition in MSW is met, it's variants are :
  • je : (jump when equal)
  • jne :(jump when not equal)
  • jz : (jump when last result was zero)
  • jg : (jump when greater than)
  • jge :(jump when greater than or equal to)
  • jl : (jump when less than)
  • jle :(jump when less than or equal to)

Note : the memory stack is the allocated amount of memory to the program, (usually from the RAM) as the name suggest the memory is "stacked" atop itself in a LIFO form (Last in,First out)
aka: push puts AX as the last in the stack and when pop take AX as the first out.
