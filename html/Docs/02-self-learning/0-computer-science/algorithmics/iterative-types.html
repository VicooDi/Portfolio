II - Linked-List :

1) Definition :

A linked is a connected array of nodes, with each nodes having the data inside the node itself and pointer to the next node in the list.
A liked-list is technically a variation of a array as most iterative types are, although it's such a complexe topic it deserves it's own header.

2) Properties :

  • Create :
    some languages come pre-built with List-Node support but even in ones that don't (like c++) creating them is simple requiring at a minimum a struct with 2 members.
struct ListNode {
	//members
	int data; //data element
	ListNode *next; //pointer to next node

	//Constructors
	ListNode() : val(NULL), next(nullptr) {}
	ListNode(int x) : val(x), next(nullptr) {}
	ListNode(int x, ListNode *next) : val(x), next(next) {}
};
  • Get :
    to get a specific element you must iterate the entier list until you find you're target element, having at worst a TC of O(n).
int Find(int target, ListNode* head){
	while(head != nullptr && head->data != target){
		head = head->next;
	}
	return head->data; //returns NULL if target doesn't exist
}
  • Assign :
    assign a value to a node at a certain index in the tree.
    requires iterating through at most the entier node list until reaching a target index making it's TC O(n)
void Assign(int val, int index, ListNode* head){
int i = 0;
	while(head != nullptr && index != i){
		head = head->next;
		i++;
	}
	if(head != nullptr){
		head->data = val;
	}
	else{
		cout<<"index out of range!"<<endl;
	}
}
  • Insert :
    Insert a specific value at a certain index in the node list

    the desired result can be achieved by iterating until reaching the target index then storing the node-to-be-replaced in a dummy that keeps permutating while iterating till the end of the nodeList.
    TC is O(n)
void inset(ListNode* head, int index, int to_insert){
	ListNode* iterator = head;
	int i = 0;
	while(iterator-next && i < index){
		iterator = iterator->next;
		i++;
	}
	
	ListNode* temp;
	while(iterator){
		temp = iterator;
		iterator.data = to_insert;
		iterator = iterator->next; 
		to_insert = temp->data;
	}
}

  • Erase :
    Erasing in a Linked List is different than a normal iterative type, there is no built in function for it in it has the risk of leaving garbage memory if not handled well. It is achieved by 3 methods depending on the position of the node-to-delete in the node list.
  • Delete from first node :
    simply set head = head->next and free the data one the
    to-be-deleted adresse. TC is O(1)
void erase_from-first(ListNode* head){
	temp = head;
	head = head->next;
	delete temp; //or free(temp)
}
  • Delete from last node :
    iterate a dummy towards the end then free it's memory it
    another method is to iterate the head itself until the end then override the last value and then freeing first node.
    TC is O(n)
void erase_from-last1(ListNode* head){
	temp = head;
	while(temp->next){
		temp = temp->next;	
	}
	delete temp; //or free(temp)
}

void erase_from-last2(ListNode* head){
	temp = head;
	head = head->next;
	delete temp; //or free(temp)
	while(head->next){
		head = head->next;
	}
}
  • Delete from a middle node :
    iterate with a dummy until reaching the desired index then
    set iterator->next = iterator->next->next; then make sure to free the unused memory.
    TC is O(n)
void erase_from-midlle1(ListNode* head, int index){
	ListNode* iterator = head;
	for(int i = 0; i < index; i++){
		iterator = temp->next;
	}
	ListNode* temp = iterator-next;
	iterator->next = iterator->next->next;
	delete temp;
}
